USB

Raw WebUSB -- no standardized interface to resolve a name against (unlike BLE GATT), so each entry names its own interfaceNumber and endpointNumber explicitly. One click performs one transferIn read; there's no generic push mechanism for raw USB, so re-click to read again.

HID (USB or Bluetooth)

WebHID reaches HID-class devices regardless of transport -- this is also the only way a browser can reach a Bluetooth HID device at all, since Web Bluetooth deliberately blocks the standard HID-over-GATT service. Live updates flow via the inputreport event, same one-click-then-live-updates shape as the BLE section.
